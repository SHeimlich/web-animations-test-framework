Web Animations Tutorial

Web animations is to have a flash or GIF animation intergrated onto the web page. Normal web animations still require the use of plugins such as Java Applets, Shockwave, Adobe Flash. In this tutorial, we will be showing you how to create animations using javascript without the need of installing plugins.


Create a new basic animation

To create animations on objects such as moving left, right, up, down, change colours, etc, use the following interface:

new Animation(element, {effect(s)}, {optional: timed items});

'new Animation' creates an animation object, 'element' is the DOM object or animation target that we want to animate and it is a compulsory component and can be nullable. For instance, 'element' can be a 'div' with, a 'p' (paragraph).

'effects' defines the types of effects in animation such as move left, or right, or change colours, or change opacity, etc. 'effects' is also a compulsory component as well as nullable. This parameter can be of AnimationEffect object or CustomAnimationEffect object. These effects will be shared with any other animation objects referring to the same AnimationEffect or CustomAnimationEffect object. In the case this component being null, the animation will have a null effect.

'timed items' can be of type double or timing dictionary and is nullable and optional component. When this parameter is double, then it specifies the duartion of a single iteration of the animation. If it is null, then the default value for iteration duration would be zero is specified in the Timing Dictionary.


Exercise 1

Now let's code some animation out! In the 'Try It Yourself' block, you should see a default div block with class named 'test' and ID named 'a' coloured in red. If you don't like the colour, you can always change it around. Click on 'Update' when you have finished typing in your code.

Now we want to make the red rounded rectangle move to 300px from left for 2 seconds. We should do the following:

new Animation(document.querySelector("#a"), {left: "300px"}, 2);
  1. 'new Animation()' creates a new animation object.
  2. 'document.querySelector("#a")' is a DOM method which gets the element with id equals to 'a' from the HTML file. You can always use 'document.getElementById("a")' which would result the same thing. Since we have defined its class name as well, we can also select it using 'document.querySelector(".test")'.
  3. '{left: "300px"}' is the effects of the animation. In this case we are letting it move to 300px from left.
  4. '2' will set the duration of the animation to 2 seconds. The bigger the number, the slower the animation speed and vice versa.
Exercise 2

Now try move the red rectangle 500px from top for 2 seconds. Try it one time without replacing the original code you have entered, then try another time with all old javascript code replaced. What do you see?


You might wonder why does the rectangle moves diagonally when you have 2 new Animation() commands. That is because you have made 2 animation objects at the same time and let them ran at the same time, if you want the movement in sequence, then you will have to apply a 'startDelay' to the timing as follow:

new Animation(document.querySelector("#a"), {left: "300px"}, 2.0);
new Animation(document.querySelector("#a"), {top: "500px"}, {startDelay: 2, duration: 2});

Notice how on the second animation object, we have put 'startDelay' and 'duration' in {}, that is because we are creating a timing dictionary to apply to the animation. Any timing dictionary will have to be stored in {}. You might also notice that number '2' is gone! That is because we have apply number '2' to 'duration' which is the duration of the animation.

Timing Dictionary and Timing Functions

In the previous section, we have learned a bit about timing dictionary. This section will explain timing dictionary details as well as introducing you to timing functions.

Timing dictionary is just a set of timing properties putting together to be applied to the animation. There are 9 members that you can include in a timing dictionary. You do not have to include all of them. As mentioned before, timing in the animation interface is optional.

    The following are the 9 members:
  • startTime
  • startDelay
  • fillMode
  • iterationStart
  • iterationCount
  • iterationDuration
  • defaultPlaybackRate
  • direction
  • timingFunction
